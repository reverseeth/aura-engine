# Competitor Analysis Engine

## Quando Usar
Quando o membro tem produto definido e market research feito, e precisa mapear o cenário competitivo antes de criar oferta e copy.

## Antes de Começar
1. Leia /workspace/profile.md
2. Leia /workspace/[produto]/01-product-research.md (se existir)
3. Leia /workspace/[produto]/02-market-research.md (se existir — tem overview competitivo básico)
4. Consulte a base Aura com as seguintes buscas (deep=true):
   - "competitor research extracting claims process"
   - "cloaker breaking competitive espionage funnels"
   - "reconnaissance engine competitive research social"
   - "market sophistication stages saturation claims"
   - "swipe file method collection organization"
   - "ad angles concepts how to create"
5. Internalize os frameworks ANTES de começar

## Fluxo da Skill

### ETAPA 1 — Identificar Concorrentes

Se o product research já identificou concorrentes, use essa lista. Se não, pergunte:

"Você já sabe quem são seus concorrentes? Se sim, me manda os links das lojas deles. Se não sabe, diga 'não sei' — o sistema encontra sozinho."

**SE o membro mandar links:** use como base e complemente se for menos de 5.

**SE o membro disser que não sabe:** pesquise automaticamente (web search: `"[produto] brand site:.com"`, `"[produto] shop"`, Meta Ad Library). Identifique 5-10 concorrentes. Se encontrar menos de 5, amplie a busca pra produtos adjacentes que resolvem o mesmo problema.

### ETAPA 1B — Ads dos Concorrentes

Verifique no `/workspace/profile.md` se o membro tem SpyBox.

**SE TEM SpyBox / Adsparo:**
"Cola screenshots dos ads mais escalados dos concorrentes — se tiver acesso ao SpyBox ou Adsparo. Se não tiver esses screenshots em mãos agora, tudo bem: sigo com o Meta Ad Library público."

**SE NÃO TEM:** pule a pergunta, use direto o Meta Ad Library público na Etapa 3.

### ETAPA 2 — Análise de PDPs dos Concorrentes

Para cada concorrente, acesse a página de produto (use web search/fetch):

Analise usando os frameworks da vault (consulte a base Aura: "competitor research extracting claims"):

**Estrutura da página:**
- Qual tipo de hero section usa?
- Qual headline principal? (copie o texto exato)
- Qual sub-headline?
- Como apresenta o produto? (foto, vídeo, GIF, demo)
- Quais bullet points de benefício usa?
- Tem mecanismo único? Qual? Como apresenta?
- Tem stack de valor visual? Quantos itens?
- Qual preço? Tem bundles? Quais?
- Tem garantia? Qual tipo?
- Tem social proof? Que tipo? (reviews, UGC, mídia, certificações)
- Tem FAQ? Quais perguntas aborda?
- Quantos CTAs? Que copy usa nos botões?
- Oferece frete grátis? A partir de quanto?

**Copy analysis:**
- Consulte a base Aura: "headlines formulas analysis" e "leads types Schwartz"
- Qual tipo de lead estão usando? (story, secret, proclamation, problem-solution)
- Qual nível de consciência a página assume do visitante?
- Quais gatilhos de persuasão usa? (escassez, autoridade, prova social, reciprocidade)
- Qual a grande promessa?
- Quais objeções a página tenta quebrar?
- Qual tom de voz?

### ETAPA 3 — Análise de Ads no Meta Ad Library

Para cada concorrente, pesquise no Meta Ad Library (use web search e Playwright quando possível):

**Métricas quantitativas:**
- Total de ads ativos
- Agrupe criativos iguais ou quase iguais — conte aparições (mais aparições = mais ad sets = mais escalado)
- Identifique os top 10 criativos mais escalados por quantidade de aparições

**Análise qualitativa dos top criativos:**
Para cada um dos top 5-10 criativos mais escalados:

- Tipo: imagem estática, vídeo UGC, vídeo demonstração, antes/depois, carrossel
- Se vídeo: transcreva com descrição do que acontece visualmente
- Hook (primeiros 3 segundos ou primeira frase): copie o texto/fala exata
- Ângulo: qual abordagem? (problema, resultado, curiosidade, autoridade, comparação)
- Copy do ad (primary text): copie o texto
- CTA: qual e como está formulado
- Landing page: pra onde manda? (PDP, landing page, advertorial)

Consulte a base Aura: "ad angles concepts how to create variations" pra classificar os ângulos.

**Padrões de formato:**
- Qual formato domina? (vídeo vs imagem vs carrossel)
- Qual duração média dos vídeos?
- Usam UGC? Que estilo?
- Usam AI voiceover ou pessoa real?

### ETAPA 4 — Análise de Claims e Posicionamento

Consulte a base Aura: "market sophistication stages claims saturation positioning"

Compile TODOS os claims que os concorrentes fazem:

**Claims diretos:** (ex: "reduz rugas em 30 dias", "resultados visíveis em 1 semana")
**Claims de mecanismo:** (ex: "tecnologia de micro-corrente", "infusão de ácido hialurônico")
**Claims de autoridade:** (ex: "recomendado por dermatologistas", "aprovado pela FDA")
**Claims de prova social:** (ex: "50.000+ clientes satisfeitas", "4.8 estrelas")

Classifique cada claim:
- SATURADO (todo mundo usa) → evitar
- COMUM (maioria usa) → usar com diferenciação
- RARO (poucos usam) → oportunidade
- AUSENTE (ninguém usa) → oportunidade forte

### ETAPA 5 — Gap Analysis

A parte mais valiosa da análise. Identifique:

**Gaps de público:**
- Existe segmento que NENHUM concorrente aborda? (ex: todos falam com mulheres 25-35, ninguém fala com 45+)
- Existe situação/trigger event que ninguém explora? (ex: pré-casamento, pós-parto)

**Gaps de messaging:**
- Existe dor real do consumidor (do market research) que nenhuma PDP aborda?
- Existe objeção que ninguém quebra?
- Existe benefício que ninguém destaca?

**Gaps de formato:**
- Todo mundo usa UGC mas ninguém usa demonstração close-up?
- Todo mundo usa vídeo mas ninguém usa carrossel educativo?
- Todo mundo usa hook de problema mas ninguém usa hook de resultado?

**Gaps de oferta:**
- Ninguém oferece garantia forte?
- Ninguém faz bundle com produto complementar?
- Ninguém tem subscription/refill?

**Gaps de mecanismo:**
- Consulte a base Aura: "unique mechanism UMP UMS differentiation new mechanism"
- Todos usam o mesmo mecanismo genérico? Qual?
- Existe espaço pra criar mecanismo proprietário?
- Existe ingrediente/feature/processo que pode ser renomeado/reposicionado?

### ETAPA 6 — Síntese Estratégica

Compile tudo num relatório estratégico:

**1. Mapa Competitivo:**
Tabela com cada concorrente: nome, preço, mecanismo, claim principal, pontos fortes, pontos fracos

**2. Padrões do Mercado:**
- O que TODOS fazem (baseline — você precisa fazer no mínimo isso)
- O que a MAIORIA faz (tendência — considere fazer mas com twist)
- Claims saturados (EVITAR)

**3. Oportunidades de Diferenciação (ranqueadas por potencial):**
- Gap #1: [descrição + como explorar]
- Gap #2: [descrição + como explorar]
- Gap #3: [descrição + como explorar]

**4. Recomendação de Posicionamento:**
- Baseado em toda a análise, como este produto deve se posicionar pra se diferenciar?
- Qual mecanismo único perseguir?
- Qual avatar focar?
- Qual ângulo principal de comunicação?

**5. Swipe File:**
- Top 3 elementos dos concorrentes que vale ADAPTAR (não copiar — adaptar o princípio)
- Top 3 elementos que NÃO vale seguir (saturados ou fracos)
